  # 后端API接口文档

## 业务逻辑

### 用户角色与权限

系统包含以下用户角色，每个角色具有不同的权限：

1. **管理员(ROLE_ADMIN)**
   - 可以管理所有用户（查看、封禁、激活、删除）
   - 可以管理所有文章（查看、发布、拒绝、删除）
   - 可以管理所有评论（查看、审核、删除）
   - 可以管理分类和标签（创建、编辑、删除）
   - 可以访问管理后台

2. **作者(ROLE_AUTHOR)**
   - 可以创建文章（初始状态为"草稿"）
   - 可以编辑自己的文章
   - 可以将自己的文章从"草稿"提交为"待审核"状态
   - 不能直接将文章设为"发布"状态
   - 可以查看自己的所有文章（无论状态）
   - 可以发表评论和点赞

3. **普通用户(ROLE_USER)**
   - 可以查看已发布的文章
   - 可以发表评论和点赞
   - 可以收藏文章
   - 可以编辑自己的个人资料

4. **访客(ROLE_GUEST)**
   - 可以查看已发布的文章
   - 不能发表评论或点赞
   - 不能收藏文章

### 文章状态流转

文章具有以下状态，状态之间的转换遵循特定规则：

1. **草稿(draft)**
   - 文章初始状态
   - 作者可以编辑、保存
   - 作者可以将其提交为"待审核"状态
   - 只有作者和管理员可以查看

2. **待审核(pending)**
   - 作者提交审核后的状态
   - 作者不能再编辑
   - 管理员可以审核
   - 管理员可以将其设为"发布"状态或拒绝回"草稿"状态
   - 只有作者和管理员可以查看

3. **已发布(published)**
   - 管理员审核通过后的状态
   - 所有用户可以查看
   - 作者不能直接修改，需要先联系管理员将状态改回
   - 管理员可以将其设回"草稿"状态或直接删除

### 文章审核流程

1. 作者创建文章，状态为"草稿"
2. 作者完成编辑后，提交审核，状态变为"待审核"
3. 管理员在管理后台查看待审核文章列表
4. 管理员审核文章内容：
   - 审核通过：将状态设为"已发布"
   - 审核不通过：将状态设回"草稿"，作者可以修改后再次提交

### 评论审核机制

1. 用户发表评论，初始状态为"待审核"或"已通过"（根据系统设置）
2. 如需审核，管理员在后台查看待审核评论
3. 管理员可以：
   - 通过评论：评论将显示在文章页面
   - 拒绝评论：评论将不被显示
   - 删除评论：评论被永久删除

## 响应格式
所有API接口返回的响应格式统一如下：

```json
{
  "success": true/false,
  "message": "响应消息",
  "data": 响应数据对象（根据接口不同而变化）
}
```

## 认证接口
基础路径：`/api/auth`

### 登录
- **URL**: `/api/auth/login`
- **方法**: POST
- **描述**: 验证用户凭据并返回JWT令牌
- **权限**: 所有用户可访问
- **请求体**:
  ```json
  {
    "username": "用户名",
    "password": "密码"
  }
  ```
- **响应**:
  ```json
  {
    "success": true,
    "message": "登录成功",
    "data": {
      "token": "JWT令牌",
      "id": 用户ID,
      "username": "用户名",
      "email": "用户邮箱",
      "role": "用户角色",
      "avatar": "用户头像URL"
    }
  }
  ```

### 注册
- **URL**: `/api/auth/register`
- **方法**: POST
- **描述**: 创建新用户账号
- **权限**: 所有用户可访问
- **请求体**:
  ```json
  {
    "username": "用户名",
    "email": "邮箱",
    "password": "密码",
    "nickname": "昵称"（可选，默认使用用户名）,
    "avatar": "头像URL"（可选）
  }
  ```
- **响应**:
  ```json
  {
    "success": true,
    "message": "用户注册成功",
    "data": null
  }
  ```

### 验证令牌
- **URL**: `/api/auth/validate`
- **方法**: GET
- **描述**: 验证JWT令牌是否有效
- **权限**: 登录用户可访问
- **请求头**: 需要包含有效的JWT令牌
- **响应**:
  ```json
  {
    "success": true,
    "message": "令牌有效",
    "data": "令牌有效"
  }
  ```

## 文章接口
基础路径：`/api/articles`

### 获取文章详情
- **URL**: `/api/articles/{id}`
- **方法**: GET
- **描述**: 通过ID获取文章详情
- **权限**: 所有用户可获取已发布文章，作者可获取自己的文章，管理员可获取所有文章
- **响应**: 返回文章详情对象

### 通过slug获取文章
- **URL**: `/api/articles/slug/{slug}`
- **方法**: GET
- **描述**: 通过slug（URL友好的标识）获取文章
- **权限**: 所有用户可获取已发布文章，作者可获取自己的文章，管理员可获取所有文章
- **响应**: 返回文章详情对象

### 获取已发布文章列表
- **URL**: `/api/articles`
- **方法**: GET
- **描述**: 获取已发布的文章列表，支持分页
- **权限**: 所有用户可访问
- **参数**:
  - `page`: 页码，默认为0
  - `size`: 每页大小，默认为10
- **响应**: 返回文章列表

### 搜索文章
- **URL**: `/api/articles/search`
- **方法**: GET
- **描述**: 搜索文章，支持分页
- **权限**: 所有用户可访问（仅搜索已发布文章）
- **参数**:
  - `keyword`: 搜索关键词
  - `page`: 页码，默认为0
  - `size`: 每页大小，默认为10
- **响应**: 返回符合关键词的文章列表

### 获取指定分类的文章
- **URL**: `/api/articles/category/{categoryId}`
- **方法**: GET
- **描述**: 获取指定分类下的文章列表，支持分页
- **参数**:
  - `categoryId`: 分类ID
  - `page`: 页码，默认为0
  - `size`: 每页大小，默认为10
- **响应**: 返回指定分类下的文章列表

### 获取指定标签的文章
- **URL**: `/api/articles/tag/{tagId}`
- **方法**: GET
- **描述**: 获取包含指定标签的文章列表，支持分页
- **参数**:
  - `tagId`: 标签ID
  - `page`: 页码，默认为0
  - `size`: 每页大小，默认为10
- **响应**: 返回包含指定标签的文章列表

### 获取指定作者的文章
- **URL**: `/api/articles/author/{authorId}`
- **方法**: GET
- **描述**: 获取指定作者的文章列表，支持分页
- **参数**:
  - `authorId`: 作者ID
  - `page`: 页码，默认为0
  - `size`: 每页大小，默认为10
- **响应**: 返回指定作者的文章列表

### 获取置顶文章
- **URL**: `/api/articles/top`
- **方法**: GET
- **描述**: 获取置顶文章列表
- **响应**: 返回置顶文章列表

### 获取特色文章
- **URL**: `/api/articles/featured`
- **方法**: GET
- **描述**: 获取特色文章列表
- **响应**: 返回特色文章列表

### 创建文章
- **URL**: `/api/articles`
- **方法**: POST
- **描述**: 创建新文章（默认状态为草稿）
- **权限**: 作者和管理员可访问
- **参数**:
  - `authorId`: 作者ID
  - `categoryId`: 分类ID
- **请求体**: 文章对象
- **响应**: 返回创建的文章对象

### 更新文章
- **URL**: `/api/articles/{id}`
- **方法**: PUT
- **描述**: 更新文章内容（仅草稿状态可更新）
- **权限**: 文章作者和管理员可访问
- **参数**:
  - `categoryId`: 分类ID（可选）
- **请求体**: 更新的文章信息
- **响应**: 返回更新后的文章对象

### 发布文章
- **URL**: `/api/articles/{id}/publish`
- **方法**: PATCH
- **描述**: 将文章状态更改为已发布，并设置发布时间
- **权限**: 仅管理员可访问，普通作者无权限调用此接口
- **响应**: 返回发布后的文章对象

### 提交文章审核
- **URL**: `/api/articles/{id}/submit`
- **方法**: PATCH
- **描述**: 将文章状态从草稿更改为待审核
- **权限**: 文章作者和管理员可访问
- **响应**: 返回待审核状态的文章对象

### 拒绝文章审核
- **URL**: `/api/articles/{id}/reject`
- **方法**: PATCH
- **描述**: 将文章状态更改为草稿
- **响应**: 返回拒绝后的文章对象

### 删除文章
- **URL**: `/api/articles/{id}`
- **方法**: DELETE
- **描述**: 删除文章
- **响应**: 返回删除操作的结果
