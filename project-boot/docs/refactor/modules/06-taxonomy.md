# 06 分类与标签模块

## 目标

- 公共用户只能读取已批准分类和标签。
- 创建、修改、删除和审核操作按产品策略限制为管理员或提交审核流程。
- 分类状态和标签状态使用明确枚举，移除 `1/0/2/PENDING` 混合表达。
- 删除分类前显式处理子分类和文章归属。

## 验证

覆盖分类树、重复名称、非法状态转换、使用中的分类删除和管理员权限。

## 已实施

- 新增分类和标签请求 DTO 与 `TaxonomyApplicationService`，写操作从 Controller 收敛到事务服务。
- 公共查询只暴露已批准分类和标签；登录用户可提交待审核项，修改和删除仅管理员可执行。
- 创建与更新校验重复名称；分类父节点必须存在且不能指向自身。

## 剩余风险

- 现有枚举/数据库状态历史值尚未通过迁移脚本统一；删除使用中的分类、父子级联策略需在产品决策后固化。
