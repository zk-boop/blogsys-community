# 05 点赞与收藏模块

## 目标

- 用户身份只来自认证上下文。
- 点赞与收藏命令幂等，数据库唯一约束作为最终防线。
- 点赞记录与文章点赞计数处于同一事务，并使用原子更新或锁避免丢失更新。
- 收藏列表的公开性形成明确产品策略。

## 验证

覆盖重复请求、并发请求、取消不存在记录、匿名访问和查看其他用户收藏。

## 已实施

- 文章点赞和收藏统一由 `InteractionApplicationService` 处理，用户身份只取安全上下文。
- 点赞记录与文章冗余计数同事务写入，计数使用原子 JPQL 更新；重复点赞/收藏和重复取消按幂等成功处理。
- 仅已发布文章允许新增互动；收藏列表只能由本人或管理员读取。
- 收藏检查接口保留旧 `userId` 参数兼容，但不再信任该值，Vue 已停止发送。

## 剩余风险

- 需要在真实 MySQL 上增加并发压力测试，确认唯一约束冲突被稳定映射为幂等响应。
