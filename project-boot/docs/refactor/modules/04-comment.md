# 04 评论模块

## 目标

- 评论作者来自认证主体，禁止客户端指定身份。
- 校验父评论属于同一文章，回复目标与父链一致。
- 作者可删除自己的评论，管理员可审核或删除；普通用户不可审核。
- 评论创建和删除与文章评论计数保持事务一致。

## 验证

覆盖匿名评论、身份冒充、跨文章回复、作者删除、管理员审核和并发计数。

## 已实施

- 新增评论请求 DTO 与事务型应用服务，评论作者固定来自当前安全主体；旧 `userId` 仅兼容接收并被忽略，Vue 已停止发送。
- 只允许对已发布文章评论；父评论和回复目标必须属于同一文章。
- 创建、审核、拒绝、删除与文章评论计数处于同一事务，计数使用数据库原子更新并防止减为负数。
- 普通用户只能删除自己的评论，审核接口仅管理员可用；公开查询仅返回已批准内容。

## 剩余风险

- 当前数据模型没有“用户-评论点赞”关联表，无法彻底保证评论重复点赞幂等；下一轮需新增唯一约束和迁移脚本。
