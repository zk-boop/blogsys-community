# 03 文章模块

## 目标

- 创建文章时作者来自认证主体。
- 草稿编辑、删除、提交审核只允许作者或明确授权管理员。
- 发布、拒绝、推荐和置顶只允许管理员。
- 文章与标签关联在单一事务中更新。

## 兼容策略

迁移期可接受旧 `authorId` 参数，但服务端忽略其身份声明并使用当前主体；Vue API 更新后移除该参数。

## 验证

覆盖匿名创建、冒充作者、非作者编辑、作者提交、管理员审核、标签更新回滚和重复 slug。

## 已实施

- 新增 `ArticleWriteRequest` 和事务型 `ArticleApplicationService`，Controller 不再直接编排 Repository。
- 创建者只取当前安全主体；兼容接收旧 `authorId`，但服务端忽略该身份声明，Vue 已停止发送。
- 草稿和待审核文章仅作者或管理员可见；他人的作者文章列表只返回已发布内容。
- 创建、更新、标签替换、状态流转和删除均在服务层校验所有权与管理员权限。
- slug 由服务端生成并执行唯一化，中文标题无可用字符时使用稳定回退前缀。

## 剩余风险

- 标签替换仍需补充数据库故障注入回滚测试；推荐/置顶的状态机还应进一步从后台 Controller 收敛到领域策略。
