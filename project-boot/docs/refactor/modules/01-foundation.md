# 01 公共基础与测试模块

## 模块目标

- 建立不依赖外部服务的可复现测试基线。
- 清除仓库内测试数据库凭据及无效 JWT 配置残留。
- 为后续安全和业务模块提供稳定回归环境。

## 本轮变更

- 测试数据库从共享远程 MySQL 改为 H2 内存数据库的 MySQL 兼容模式。
- 添加最小、确定性的测试数据集，不再依赖生产式 SQL 导出和固定外部状态。
- 服务测试禁用 JDBC Session 持久化；Session 行为将在认证模块通过专项集成测试覆盖。
- 主配置不再提供默认数据库密码，部署环境通过 `DB_PASSWORD` 注入。
- 测试文件与上传目录落到 `target/test-uploads`。

## 接口与数据影响

- 生产 API：无变化。
- 生产数据库结构：无变化。
- 测试数据：仅存在于进程内存，测试结束即销毁。
- 环境变量：本地数据库存在密码时必须设置 `DB_PASSWORD`。

## 验证命令

```powershell
$env:JAVA_HOME='C:\Program Files\Java\jdk1.8.0_202'
mvn clean test
mvn package -DskipTests
```

## 完成标准

- 所有测试无需互联网或共享数据库即可启动。
- Maven 测试与打包通过。
- 测试输出不包含数据库密码、Cookie 或 Session ID。

## 验证结果

- 初始基线 `mvn clean test`：24 项测试通过；完成各模块后全量为 34 项，0 failure，0 error。
- 测试不再访问互联网或共享数据库。
- 中文种子数据通过显式 UTF-8 初始化。
- 增加全局异常映射，隐藏未预期异常和数据库内部信息。
